# 智能多元化互动空间：AI 问答平台 · 像素拼豆 · Nexus 3D 数字人系统

本项目**由作者之前开发的“智能问答系统”衍生与进化而来**。在承接并深拓原问答系统的基础上，现已演进为一套高度集成、响应式、模块间深度协作的现代多元化 AI 综合应用平台。目前系统已完成一系列重磅的升级与优化，完美融合了文本生成、图像编辑、3D 渲染与超拟真语音对话。

---

## 🌟 核心系统模块

### 1. 智能问答与智慧教育系统 (AI Q&A & Edu Platform)
*   **多领域智能问答**：集成电影专项问答、通用知识图谱问答（Neo4j）、深层语义相似度检索。
*   **AI 课件大纲设计**：在“功能体验”模块提供一键结构化 PPT 大纲规划功能，并支持设定风格（古典雅致/现代科技/简约学术/活泼卡通）一键生成 PPT 配音讲解稿。
*   **引擎底层升级**：将大纲与课件生成的 AI 底座由容易超时的旧模型统一升级为模力方舟（Moark）**`DeepSeek-V3`** 模型，并将请求超时宽限提升至 **120秒**，彻底消除超时卡死问题。
*   **全屏系统管理**：宽屏自适应布局，直观展示服务连通性、数据库状态与模型参数。

### 2. 像素拼豆工作台 (Bean-to-Gold Studio)
*   **AI 文本/上传图片转拼豆**：输入画面创意描述或上传本地图片，自动转换为 16x16 / 32x32 / 64x64 全彩高亮像素拼豆图样网格。
*   **👤 一键应用为 2D 数字人模型**：拼豆图样下方提供“应用为 2D 数字人脸”功能，一键将设计出的像素形象同步覆盖至 2D 数字人模型，并可生成嘴型同步。
*   **💾 多格式导出功能（PNG/JPG/SVG/PDF）**：
    *   **PNG**：无损透明高清图纸。
    *   **JPG**：自动背景垫白的高清晰度压缩格式。
    *   **SVG**：矢量的无限等比缩放设计图纸。
    *   **PDF**：智能适应宽高比并一键打包导出的电子文档版本（自动动态加载 `jsPDF` 库）。

### 3. Nexus 3D 数字人系统 (3D Digital Human Sandbox)
*   **实时 3D 渲染**：支持拖拽上传 `.glb` 模型、`.obj`、`.3mf` 以及 `.jpg` 三维背景资产。
*   **🎙️ 语音引擎升级 (Fish-Speech)**：
    *   彻底移除了本地配置复杂的 GPT-SoVITS 依赖，升级为模力方舟提供的 **`fish-speech-1.2-sft`** 极速 API 驱动，声音表现生动且无本地 CPU/GPU 卡顿。
    *   **音频格式自适应**：前端 [Web3DScene.jsx](file:///c:/Users/Asus/Desktop/gooduse/frontend2/src/components/Web3DScene.jsx) 新增 Base64 字节特征流自动探测（WAV vs MP3），彻底修复了多平台解码报错。
*   **动作反馈与状态同步**：实现 3D 虚拟人动作（Wave/Talk/Dance/Jump）的实时 Websocket 广播与状态双向同步。

## 🚀 极速启动指南

### 1. 环境准备
*   **Python**：推荐使用 Python 3.11+，并配置激活在根目录的 `venv` 虚拟环境中。
*   **Node.js**：推荐 Node.js 18+ 以运行 Vite 前端。
*   **数据库**：MySQL 8.0+（必要）及可选 Neo4j 图数据库。

### 2. 依赖安装

```bash
# 1. 激活 Python 虚拟环境并安装依赖
venv\Scripts\activate
pip install -r requirements.txt

# 2. 安装拼豆前端依赖
cd bean-to-gold/frontend
npm install

# 3. 安装 3D 数字人前端依赖
cd ../../frontend2
npm install
```

### 3. 一键启动

系统提供了一键启动 PowerShell 交互管理脚本，省去繁琐的手动开启步骤：
```powershell
.\start_project.ps1
```
运行后，按照控制台交互菜单输入对应数字即可自动批量拉起所有服务（主系统 5001、统一后端 8085、交互 WebSocket 8765、语音 8000 以及两个前端）。

## 🛠️ 常见问题与解决方案

### 问题 1：本地语音生成（TTS）环境依赖重、运行卡顿
*   **问题描述**：3D 数字人原先依赖本地 `GPT-SoVITS` 模型，部署需要极其复杂的 PyTorch/CUDA 环境，且普通 CPU 推理延迟长达 15 秒，导致对话极其卡顿。
*   **解决方案**：将语音引擎替换为云端极速 **`fish-speech-1.2-sft`** API，并在前端增加 Base64 音频头特征自适应识别，实现 `WAV/MP3` 双格式秒级解码播放，响应缩短至 1 秒以内。

### 问题 2：原问答机器人仅支持本地知识库查询，无法解答库外通用问题
*   **问题描述**：原版智能问答系统只能死板地比对本地 Neo4j/MySQL 知识库，一旦用户提问超出预设库的范围（如通识问题、推理任务等），系统便无法给出任何合理回答。
*   **解决方案**：重构并接入了主流大语言模型 API 接口，并引入了 RAG（检索增强生成）模式。系统在运行时会先在本地数据库检索相关事实，随后将检索结果注入到大模型的上下文 prompt 中一同提交。这不仅保留了本地专业知识的精确度，又让系统具备了强大的通用推理与回答能力。
