公网地址可访问，登录、问答机器人、DeepSeek 回答、讨论区、注册页、登出保护等主路径基本可用。当前主要问题集中在公网部署版本与本地最新版本不一致、部分交互缺少可见反馈、数字人状态不复位，以及普通用户访问管理员页面时体验不友好。

## 问题列表

### P1：公网版本缺少 DeepSeek API Key 界面配置框

现象：问答机器人页面有回答模式下拉框，但没有 `DeepSeek API Key` 输入框。

期望：页面应显示本地新版本中新增的 `DeepSeek API Key` 输入框、保存按钮和清除按钮，支持用户在界面中配置 API Key。

影响：用户无法在公网界面自行配置 API Key，只能依赖服务器环境变量。与“把 apikey 的配置做到软件界面里”的需求不一致。

复现步骤：

1. 打开 `https://myqa.cpolar.top/qa_robot`。
2. 使用账号登录。
3. 查看左侧提问区域。
4. 观察不到 `DeepSeek API Key` 输入框。

建议处理：将本地最新代码同步到公网服务器，并重启 Flask/cpolar 映射服务。

### P1：数字人回答后状态长时间停留在“正在思考并合成语音...”

现象：数字人页面提交文本问题后，系统返回了文字答案，但页面状态在 70 秒后仍显示 `状态: 正在思考并合成语音...`。

期望：文字回答完成或语音合成结束/失败后，状态应恢复为 `空闲`，或显示明确的 TTS 错误。

影响：用户会误以为系统仍在处理，难以判断是否已经结束，也可能影响后续交互信心。

复现步骤：

1. 登录后打开 `/digital_human`。
2. 在输入框提交：`测试：请用一句话说明你是谁。`
3. 页面返回文字答案。
4. 等待约 70 秒。
5. 状态仍显示 `正在思考并合成语音...`。

建议处理：检查数字人页面的 TTS Promise、音频播放回调、异常捕获和状态复位逻辑；无论 TTS 成功或失败，都应在 finally 中恢复状态。

### P2：讨论区空内容发布没有明显提示

现象：讨论区内容为空时点击 `发布讨论`，页面没有明显错误提示，也没有禁用提交按钮。

期望：空内容时按钮应禁用，或点击后显示“内容不能为空”。

影响：用户不知道点击是否生效，也不知道为什么没有发布。

复现步骤：

1. 登录后打开 `/qa_discussion`。
2. 保持讨论内容为空。
3. 点击 `发布讨论`。
4. 页面无明显反馈。

建议处理：前端提交前校验 textarea 内容；为空时显示可见错误提示。

### P2：普通用户访问 `/admin` 时直接显示 JSON

现象：普通用户访问 `https://myqa.cpolar.top/admin` 时，页面直接显示 JSON：

```json
{
  "error": "需要管理员权限",
  "success": false
}
```

期望：管理员页面路由应返回友好的权限不足页面，或跳转到管理员登录页。

影响：权限逻辑有效，但用户体验像接口报错页面。

复现步骤：

1. 使用普通账号登录。
2. 访问 `/admin`。
3. 页面显示原始 JSON。

建议处理：区分页面路由和 API 路由；页面路由用模板提示，API 路由返回 JSON。

### P3：问答机器人语音播放按钮缺少可见反馈

现象：点击 `输出中文` 后，没有弹出错误，也没有出现可见加载、成功或失败提示；页面中也没有新增音频控件。

期望：点击后应显示“正在生成语音/播放中/播放失败”等状态。

影响：如果用户没有听到声音，无法判断是按钮未生效、浏览器禁止播放、TTS失败，还是正在后台处理。

复现步骤：

1. 登录后进入 `/qa_robot`。
2. 提交问题并获得回答。
3. 点击 `输出中文`。
4. 观察页面没有可见状态变化。

建议处理：为 TTS 按钮增加加载状态、成功状态和失败提示；若使用隐藏 Audio 对象播放，也应给用户可见反馈。

## 其他观察

1. 问答机器人 DeepSeek 回答能正常返回，但本地知识库候选结果与测试问题匹配度较低，页面仍展示候选答案。建议在相似度过低时隐藏候选或标记为“低相关”。
2. 问答机器人每次测试提问会新增历史记录；建议后续增加“测试记录清理”或管理员清理能力。
3. 讨论区列表能加载，但没有进一步执行真实发布测试，避免污染生产数据。
4. 注册页只做了页面和空表单校验检查，没有创建新账号。
5. 管理员后台没有管理员账号，本轮只验证了普通用户权限拦截。

## 建议优先级

1. 先部署本地最新版本，补上界面配置 API Key 能力。
2. 修复数字人状态不复位问题。
3. 补齐讨论区和 TTS 的前端反馈。
4. 优化 `/admin` 普通用户访问体验。
